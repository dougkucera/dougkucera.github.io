  • QUESTION:
    When would you use "=" vs "==" vs ".equals"? Explain what each of these does and in which cases you should use them.

    ANSWER:
    Each 'symbol' would seem like the same way to accomplish the same thing but each is unique and used for a different purpose.

    • "=" is called an assignment operator and used to assign a value to a variable. An example would be int x = 10; .
    • "==" is a comparison operator to compare values OTHER THAN strings. A good example would be if used in an if statement: "if x == y"
    • Similar to"==", ".equals" is the comparison operator used to compare strings. A similar example would be if used in an if statement: "if string1 == string2"

  • QUESTION:
    What does it mean to "cast a double to an int"? Give an example of when you would do this and explain what is happening.

    • ANSWER:
    Type casting is used to assign a variable type to a different type. This may be used if the original assigned variable type is not compatable in a way that you need it to be. An example would be if the original variable definition were of type double, but you needed it to be an integer (int):

    • double d = 2.0;
    • int = x, sum;
    • If you needed to sum d and x, you would need to make the variables of the same type:
    • sum = x + (int)d;

  • QUESTION:
    What is the difference between "switch" and "if/else"? When would you use one versus the other?

    ANSWER:
    The major difference between "switch" and "if/else" is how the expression is checked.

    • If/else: Walks sequentially through the condition starting with the first "if". It will work through the if/else until it finds a matching condition or reaches the end of the statement.
    • Switch: has indexed the values to be matched and will jump to the matching value or move to the last "default" case and execute/exit.

  • QUESTION:
    What is "void" in a method declaration? How does this differ from declaring a method with a return type?

    • ANSWER:
    Void in a method declaration denotes that nothing will be returned from that method. An example would be: "public void funtionName()"

    If a return value is expected, the type would be declared as so: "public static int functionName()"
